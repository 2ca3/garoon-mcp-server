# プロジェクト基本情報

- このプロジェクトは Python で書かれた MCPサーバです

# コードスタイル

- 必ず、ツール毎にユニットテストを作して下さい

## コーディング規約

### Python (Ruff適用)

- **行長**: 120文字
- **引用符**: ダブルクォート (`"`)
- **docstring形式**: NumPy スタイル
- **改行コード**: LF (Unix)
- **除外ディレクトリ**: `db_schema/` (Submodule)

**ファイル別ルール:**
- `tests/**`: assert文、マジック値、docstring省略可、日本語関数名許可、パスワード直書き許可
- `scripts/*`: print文許可

# Garoon API実装ルール

- Garoon APIに関する実装判断（エンドポイントの選択・パラメーター値・リクエスト構造など）は、必ず [cybozu.dev Garoon REST APIドキュメント](https://cybozu.dev/ja/garoon/docs/rest-api/) を確認してから行うこと
- URLが提示されない場合も、推測で実装せずドキュメントを参照する
- 既存の情報（環境変数・設定値など）で代替できる場合は、不要なAPIコールを追加しない

# ワークフロー

- 変更完了後は必ず型チェックを実行
- 変更完了後は必ずユニットテストを実行
